Editorial Method
Overview
Section summary

This guide follows four principles:

  • Rule function over flavor text. Introductory and thematic text has been trimmed unless it helps explain the system.
  • Specific wording preserved where it matters. Timing windows, target legality, costs, and restrictions are stated as directly as possible.
  • General rules first, exceptions second. This makes the game easier to learn without changing the outcome.
  • Source inconsistencies are flagged, not silently fixed. Where the source set appears to use conflicting terminology, this guide notes it.
Reader warning
Clarification is not a license to improvise

If a note explains a rule interaction, treat it as a reading aid tied to the source material, not as permission to ignore the actual restriction, trigger, or legality test in the underlying rule.

RAW audit note

This edition was checked against the project RAW source files in Parts 2–12. No dead numbered cross-references were found there. Most disputes came from wording, source artifacts, or summary files rather than broken section numbering.

The source files still contain non-rule artifacts such as "<< VISUAL >>" markers, "<a5 example>" placeholders, and one duplicated broken sentence in the Deploy text. Those were treated as layout debris rather than rules text.

Second-pass RAW audit note

A second focused audit then checked battlefield logic, timing windows, and keyword interactions that affect movement and engagement. No broken numbered references were found there either, but several rulings only become clear when Parts 4, 7, 8, 10, and 11 are read together.

The highest-friction cross-part combinations were cover versus visibility, engagement versus ranged targeting, Active versus Reaction timing windows, and the keyword set PLACE / INSTANT / DISPLACEMENT / FLYING / BURROWED.

Models, Units, and Bases
This section fixes the game’s physical baseline. A model is one miniature on one base, a unit is one or more models acting together, and the base — not the sculpt — governs measurement, contact, and legality.
Page role
Physical Rules Base

Use this page to lock down the game’s physical objects before later movement and combat exceptions start layering on top.

Read for
What Measures

Most later arguments disappear once base-to-base measurement replaces sculpt shape, pose, and scenic height.

1.1 Models
Rule text

A model is a single miniature on its own base. Rules interact with the base, not the sculpt above it. Pose, weapon angle, scenic height, and dramatic flight pose do not change how the rules work.

1.2 Units
Rule text

A unit is one or more models acting together as a single formation. A unit:

  • moves as one entity
  • attacks as one entity
  • is constrained by coherency
  • is either friendly or enemy depending on who controls it.
1.3 Bases
Rule text

The base is the true game piece.

Measurements are made base-to-base.

Bases may never overlap.

If a base cannot physically fit, the model cannot go there.

Scenic basing does not change size, line of sight, or positioning.

For models on flight stands, the bottom of the stand counts as the base.

1.4 Wobbly Model Handling
Rule text

If terrain prevents a model from standing flat, both players should agree where it would stand if it could be placed normally. The model is treated as occupying that agreed position.

RAW references: Parts 2.1-2.3, 4.1.

Tags, Player Roles, Keywords, and Rule Priority
This section sets the rule language the rest of the book uses. Tags identify what something is, player-role terms identify whose effect is resolving, and keywords or specific text tell you when a general rule stops applying.
Page role
Rules Language

This page defines the labels and hierarchy that tell you what an object is, whose effect is resolving, and when specific text overrides the baseline rule.

Read for
Meaning Before Action

Separate ordinary wording from formal keywords first, then check whether the rule you are reading is general or specific.

2.1 Faction Tags
Rule text

Faction tags determine army legality.

A faction card always has a race tag and often a sub-faction tag.

A unit card or tactical card may only be included if every tag on that card appears on the faction card.

The card does not need to match every tag on the faction card; it only needs all of its own tags to be present there.

2.2 Combat Tags
Rule text

Combat tags describe battlefield nature and targeting relevance. Examples include:

  • Ground
  • Flying
  • Armoured
  • Biological
  • Mechanical
  • Light
  • Psionic

Combat tags matter for:

  • weapon target restrictions
  • surge matching
  • many special abilities and keywords.
  • Important distinction: Ground as a combat tag is not the same thing as GROUND LEVEL as elevation.
2.3 Player Roles
Rule text

These terms matter constantly:

  • Active Player: the player whose turn it is to act.
  • Controlling Player: the player currently making decisions for a specific unit, model, or token.
  • Friendly / Enemy: determined by controller and team relationship.
2.4 Keywords
Rule text

Keywords are written in BOLD CAPS and always mean the same thing wherever they appear. Two global rules matter:

  • Keywords do not stack.
  • If a numeric keyword is gained twice, use only the highest value.
2.5 Rule Priority
Rule text

When a specific card, ability, mission rule, or other explicit text contradicts a general rule, the specific rule takes precedence.

That principle is one of the most important advanced-user tools in the game. When a ruling feels uncertain, ask:

  • What is the general rule?
  • Is there a more specific rule on the card, ability, mission, or keyword?
  • If yes, the specific text wins.
RAW references: Parts 2.4-2.6, 11.
RAW clarification note

A lowercase word is not automatically its keyword version. Read the glossary meaning only when the source is clearly invoking the formal keyword. Example: "instant" in the Supply Pool section is ordinary English, not the INSTANT weapon keyword.

II.01
Part II
Dice Language
This section gives you the game’s dice grammar. It separates rolls, tests, re-rolls, modifiers, and fixed additions so later combat and ability text can be read without guesswork.
Page role
Dice Grammar

This page builds the dice vocabulary used everywhere else, so later combat steps and ability timing are not carrying hidden assumptions.

Read for
Number Needed Vs. Value Generated

Keep target-number modifiers separate from generated values like D3+1, because they affect different parts of play.

Overview
Section summary

The game uses standard six-sided dice.

D6: roll one six-sided die.

D3: roll one D6 and halve the result, rounding up.

2D6 / 3D6: roll that many dice and add the results.

A Roll-Off is always both players rolling 2D6, with ties re-rolled.

3.1 Re-Rolls
Rule text

A re-roll replaces the original result, even if the new result is worse. Unless a rule says otherwise, a re-roll applies to one die.

3.2 Tests
Rule text

A test is a roll against a target number. There are two broad types:

  • Characteristic tests: use a unit stat such as Armour or Evade.
  • Attribute tests: use a weapon or ability value such as Hit.
  • Not every die roll is a test. Some rolls only generate a value, such as a D3 result or a Charge roll.
3.3 Modifiers
Rule text

Modifiers change the target number, not the die result.

+X modifier makes the roll easier by reducing the target number by X.

-X modifier makes the roll harder by increasing the target number by X.

Global Limits:

  • A target number cannot be improved below 2+.
  • A target number cannot be worsened above 6+.
  • A natural 6 always succeeds.
  • A natural 1 always fails.
  • A null value (-) cannot be turned into a valid roll by modifiers.
3.4 Fixed Additions
Rule text

Expressions such as D3+1 generate a value. They are not target-number modifiers.

3.5 Cocked Dice
Rule text

If a die does not land flat, or the result is not clear, re-roll it.

Measurement Basics
Start here for how distance works. Measurements are base-to-base, pre-measuring is always allowed, and elevation only matters when a rule says it does.
Common mistake
Players often measure the model, not the base

The rules object is the base. Wings, guns, scenic rocks, and model height do not create extra range by themselves unless a specific rule says otherwise.

Check yourself • Does a tall scenic base change range?

No. The base is the rules object. Scenic height and pose do not change range by themselves.

4.1 Pre-Measuring
Rule text

Players may measure any distance at any time. There is no hidden-distance play.

4.2 How To Measure
Rule text

Measure from the nearest point of one base to the nearest point of the other base.

Overhanging weapons, wings, or scenic elements do not count.

Base-to-base contact is 0".

Tokens and markers are measured to their nearest physical edge.

4.3 Elevation And Distance
Rule text

Distances between models at different elevations are measured horizontally from a top-down view. Vertical height is ignored for range, engagement range, and most ability ranges.

Quick check
Ask these before arguing about distance
Question 1

Are both players measuring from the nearest point of each base rather than from the miniature itself?

Question 2

Is the distance one of the rules cases that ignores vertical height and uses the horizontal top-down view?

Within, Wholly Within, and the Leading Model
These three terms control a huge amount of placement logic. "Within" cares about any part of the base, "wholly within" cares about the full base, and the Leading Model sets the point the rest of the unit rebuilds around.
Page role
Placement Terms

This page fixes the meaning of the most important range and movement terms before they are reused inside deployment, charge, and objective rules.

Read for
Which Part Of The Base Matters

The difference between within, wholly within, and the Leading Model controls whether a later placement is just close enough or fully legal.

5.1 Within
Rule text

A model is within a distance if any part of its base touches or crosses that range. A unit is within if at least one of its models is within.

5.2 Wholly Within
Rule text

A model is wholly within only if its entire base lies inside the range. A unit is wholly within only if all its models satisfy that condition.

5.3 The Leading Model
Rule text

Most movement is resolved through a leading model. When a unit moves, deploys, runs, charges, disengages, closes ranks, or otherwise repositions through standard movement:

  • choose a leading model
  • move that model first and measure its exact path
  • then set the remaining models into legal coherency around it.
  • The leading model is a temporary movement reference, not a permanent role.
RAW clarification note

When a later rule says a unit Moves, Runs, Charges, Deploys, Disengages, or Closes Ranks under standard move logic, carry the normal Leading Model restrictions with it unless that later rule gives a specific override. That includes path measurement, gap clearance, terrain, elevation, overlap, and legal end-position checks.

III.03

Coherency
Coherency is the rule that keeps a unit functioning as one formation after it repositions. When the move ends, every model must still connect back to the Leading Model through a legal chain.
Worked ruling
A chain is not enough by itself

If a rear model is within 3" of another model but the unit cannot trace a legal link back to the Leading Model through legal spaces, the unit is not in legal coherency at the end of the reposition.

Margin reminder
The three coherency checks

Ask these in order: distance to the Leading Model, legal link back through the unit, and whether every model can actually fit in a legal final position.

Leading ModelWholly WithinCoherency LinkLegal Final Position
Selective RAW lens
Coherency is two checks, not one
RAW says
“All models in the Unit must be Wholly Within 3\" of the Leading Model and able to trace a valid Coherency Link to it.”
Plain English

Distance to the leader and legal link back through the unit are separate requirements. A model can be close enough and still fail the link test.

Table effect

At the table, check distance first, then ask whether the chain back to the Leading Model is legal through legal spaces.

Common misread • Why is this not just chain-to-the-next-model?

Because this guide reads coherency as leader-anchored at the end of a repositioning action. The chain must be able to trace back to the Leading Model, not just to whichever model happens to be nearest.

Overview
Section summary
  • Coherency is checked when the reposition ends. Later casualties do not trigger a fresh coherency check by themselves.
  • If a model cannot be placed legally, remove it as a casualty.
Coherency Link

A Coherency Link is a legal chain through models in the same unit. It cannot pass through other units, terrain, or gaps the Leading Model could not legally move through.

Why it matters

A unit that is out of coherency cannot control or contest mission markers.

Practical reminder

Move the Leading Model first, then rebuild the rest of the unit around it.

RAW clarification note

Coherency is leader-anchored, not free daisy-chaining. At the end of a repositioning action, every model must be wholly within 3" of the Leading Model and able to trace a legal Coherency Link back to it.

Casualties by themselves do not trigger a new coherency check later. A unit only becomes Out of Coherency if the condition was not met at the end of its last repositioning action.

RAW clarification note

For non-Flying units, a Coherency Link cannot pass through models from other units, terrain, or a gap the Leading Model itself could not move through. For Flying units, the source gives a specific override: ignore terrain and allow links through other units.

Gap Clearance, Terrain Passage, and Legal Movement Space
This section is about physical fit, not intent. A move is legal only if the Leading Model can pass through the route, avoid illegal overlaps, and end where the base actually fits.
Page role
Movement Legality

This page is about whether space is actually passable, not whether a player wishes a move to happen.

Read for
Can The Base Really Fit

A theoretical route is never enough by itself. Gap size, passable openings, and physical fit all still have to line up.

7.1 Gap Clearance By Size
Rule text

The leading model can pass through terrain gaps depending on unit size:

  • Size 2 or lower: requires at least a 1" gap.
  • Size 3 or larger: requires at least a 3" gap.
  • Even if the gap is theoretically legal, the base still has to physically fit.
7.2 What Counts As A Gap
Rule text

Openings such as archways, breaches, and doorways count as gaps if both players agreed they are passable during battlefield setup.

7.3 Flying And Gap Clearance
Rule text

Flying units ignore gap-clearance restrictions during movement.

RAW references: Part 4.6, Part 8.5.3.
RAW clarification note

Gap Clearance is not only for ordinary Move actions. Charge explicitly measures along the real path and follows Gap Clearance, and Close Ranks inherits the same logic because it tells you to move the Leading Model under standard move rules.

IV.01

Unit Cards
Read unit cards as rules objects, not just stat blocks. Each field tells you something different about movement, defense, battlefield role, weapons, or timing.
Selective RAW lens
Null Speed and PLACE
RAW says
“Null Value (-): The Unit cannot move or be repositioned by any means, including PLACE and involuntary movement effects.”
Plain English

A dash in Speed is a general shutdown of movement and repositioning. It is broader than “cannot take a normal Move.”

Table effect

When a card uses PLACE, do not assume that PLACE automatically overrides Speed -. First look for a specific override that says this effect still works.

Page role
Card Reading

Use this page to understand how a unit card communicates battlefield capabilities before those values appear inside later sequences.

Read for
Field Meaning

Treat each line on the card as a rules field with a job, not as general flavor text about the unit.

Overview
Section summary

A unit card tells you:

  • Unit Name
  • Speed
  • Evade
  • Armour
  • Hit Points
  • Shields (If Present)
  • Size
  • Supply Profile
  • Phase-Limited Abilities
  • Weapon Profiles
  • Combat Tags
  • Faction Tags
8.1 Split Speed Values
Rule text

If a unit has a split speed like 4"/7", use the second value only when:

  • the unit has been reduced to one model, or
  • it started the game as a single-model unit.
8.2 Null Values
Rule text

If a characteristic shows -, that capability does not exist. Examples:

a unit with Evade - cannot make Evade rolls

a unit with Speed - cannot move or be repositioned by any means, including PLACE, unless a more specific rule overrides the general movement restriction.

8.3 Shields
Rule text

If a shield value is present:

  • the first model gains those shield points in addition to HP
  • the unit has the Shielded status while the shield remains
  • losing the shielded status does not erase the model’s normal HP.
8.4 Weapon Profiles
Rule text

Weapon profiles specify:

  • phase of use
  • range
  • valid target type
  • rate of attack (RoA)
  • hit number
  • surge type
  • surge dice
  • damage
  • keywords.
  • Melee weapons use E for engagement range. Template weapons use FT or BT.
RAW clarification note

A null Speed value does more than stop a normal Move. RAW treats Speed - as a general inability to move or be repositioned by any means, including PLACE, unless a more specific rule explicitly overrides that restriction.

Upgrades, Tactical Cards, Faction Cards, Mission Cards, and Deployment Cards
The non-unit cards do different jobs, and the differences matter. Some affect army construction, some shape setup, and some create live rules on the table.
Page role
Non-Unit Card Roles

This page separates the different jobs played by upgrades, tactical cards, faction cards, mission cards, and deployment cards.

Read for
Function By Card Type

The key is not memorizing every card at once, but knowing whether a card changes list building, setup, or live battlefield play.

9.1 Upgrades
Rule text

Unit-card backs list upgrades purchased during army building. Important upgrade language:

  • ↑ FOR [weapon] means full replacement.
  • specialist upgrades can create separate attack batches because different models now carry different weapons.
9.2 Tactical Cards
Rule text

Tactical cards are bought with Vespene Gas. They can:

  • unlock additional army slots
  • provide a faction resource when exhausted
  • grant active, passive, or reaction abilities.
9.3 Faction Cards
Rule text

Every army must include exactly one faction card. It provides:

  • faction identity
  • starting army slots
  • a faction resource type
  • faction abilities.
9.4 Mission Cards
Rule text

Mission cards define:

  • Starting Supply
  • Supply Escalation

Line of Sight
Line of sight is a visibility check, not a sculpt check. Read it from above, base to base, using terrain and effective size rather than pose or model height.
Selective RAW lens
A movement opening is not automatically a sight line
RAW says
“Unless players agree otherwise during Battlefield Setup, gaps, windows, doorways, and open interiors within the footprint do block the Line of Sight.”
Plain English

The default assumption is visually closed, not visually open. Table agreements can change that, but only if players set them in advance.

Table effect

A doorway can count as a legal movement gap while still blocking line of sight. Decide those exceptions during setup, not in the middle of an attack.

Common misread
A movement opening is not automatically a sight line

Players often assume that if a model can move through a window, breach, or doorway, it can also see through it. RAW does not do that by default. Movement passability and line-of-sight passability are separate agreements.

Overview
Section summary

To check visibility:

  • trace a straight line from any part of the attacker’s base to any part of the target’s base
  • evaluate the terrain the line crosses.
  • If the line does not pass through blocking terrain, the target is visible.
  • If the line does pass through blocking terrain, visibility depends on the cover rules.
11.1 Terrain Footprint
Rule text

For LoS, a terrain footprint is its base or the outermost edge of its solid structure viewed from above.

Unless players agreed otherwise during setup, windows, doorways, and interior openings still block line of sight if they are inside that footprint.

11.2 Blocking Terrain
Rule text

Any terrain with effective size 1 or greater is blocking terrain. Whether it truly blocks a given shot depends on the cover rules.

RAW clarification note

A passable opening for movement is not automatically open for Line of Sight. RAW says windows, doorways, gaps, and open interiors inside a terrain footprint still block LoS unless players agreed otherwise during battlefield setup.

RAW clarification note

Blocking Terrain is any terrain piece with Effective Size 1 or greater. Tokens are handled elsewhere as terrain of Size 0, so they do not become Blocking Terrain unless a more specific rule says they do.

VI.02
Part VI
Cover
Cover asks whether terrain merely lies between the models or actually changes visibility. The key split is between Full Cover and Direct Cover, and that split decides whether the target disappears or remains attackable.
Common misread
Crossing terrain is not automatically enough

The line has to cross blocking terrain, and then one specific terrain piece still has to qualify as either Full Cover or Direct Cover. If no single terrain piece meets those tests, the target remains visible.

Worked example
Full cover

If the same terrain piece is effectively as tall as both attacker and target, and the shot crosses it, line of sight is blocked outright.

Worked example
Direct cover

If the line crosses terrain and one of the models is within 1" of that terrain, check whether that one terrain piece is large enough to block through direct cover.

Check yourself • Do multiple small terrain pieces combine into one bigger cover wall?

No. Each terrain piece is tested on its own. You do not add terrain sizes together to create cover from several pieces that each fail on their own.

Return to Cover ↑
Overview
Section summary

There are two main ways terrain blocks line of sight:

12.1 Full Cover
Rule text

A terrain piece blocks line of sight if its effective size is equal to or greater than the effective size of both the attacker and the target.

12.2 Direct Cover
Rule text

If the LoS line passes through terrain and either the attacker or target is within 1" of that terrain, line of sight is blocked if the terrain’s effective size is equal to or greater than the effective size of the model that is within 1".

12.3 Close-Quarters Exception
Rule text

If both models are within 1" of the same terrain piece and within 3" of each other, direct cover does not apply. Resolve line of sight normally.

12.4 Terrain Pieces Do Not Combine
Rule text

Each terrain piece is evaluated independently. You never add the size or “cover effect” of multiple terrain pieces together to create cover.

RAW clarification note

Crossing Blocking Terrain is not enough by itself to hide a target. RAW resolves visibility in two steps: first ask whether the line crosses Blocking Terrain, then ask whether one terrain piece actually qualifies as Full Cover or Direct Cover. If no single terrain piece meets either test, the target remains Visible.

Direct Cover is also two-way through the same terrain piece. If that terrain blocks the shot, neither model has Line of Sight to the other through it. This is a visibility rule, not just a defensive bonus.

Does the LoS line cross Blocking Terrain at all?

Pass 2

If yes, does one terrain piece meet Full Cover or Direct Cover on its own?

If no

No crossing terrain means no terrain-based visibility block.

If still no

Crossing terrain without a qualifying cover test still leaves the target visible.

Engagement and Engagement Restrictions
Engagement is a formal battlefield state with downstream consequences. Once it exists, it changes movement, targeting, and who must act later in combat.
Common mistake • If two bases are within 1", are they always engaged?

No. Distance is only one gate. Terrain, flying status, and elevation restrictions can still stop engagement from existing.

Return to Engagement and Engagement Restrictions ↑
Worked ruling
Treat engagement like a gated state, not a distance shortcut

Being within 1" only opens the first gate. You still need compatible tags, a legal terrain relationship, and an allowed elevation relationship before engagement actually exists.

This is why many table disputes sound like range disputes but are really terrain or elevation disputes.

14.1 Engagement Range
Rule text

Engagement range extends 1" horizontally from any model’s base.

When any model in one unit is within 1" of any model in an enemy unit, both units are engaged, provided the engagement restrictions are satisfied.

14.2 Engagement Restrictions
Rule text

Engagement only exists when:

  • combat tags are compatible
  • terrain does not prevent engagement
  • elevation relationships allow it.

Examples:

  • Ground models engage Ground models.
  • Flying models are never engaged.
  • Size 2+ terrain blocks engagement through it.
  • High ground and ground level do not engage each other directly.
  • Mid ground can engage up or down only through the same access point link structure described in the rules.
14.3 Flying And Engagement
Rule text

Flying units:

  • are never engaged
  • never participate in the Combat Phase
  • cannot be chosen as close-combat targets.
RAW references: Part 7.
RAW clarification note

The MID GROUND access-point clause is a likely dispute point. When terrain uses linked access points, agree during setup which access point network a model is adjacent to so later engagement checks stay consistent.

Engagement is also tag-locked. Ground models Engage Ground models only. Flying models are never Engaged.

Size 2+ terrain can block engagement even when the models are within 1" horizontally. Proximity by itself is not enough if terrain sits between them in a way the engagement rules forbid.

Flying changes engagement only. It does not cancel ordinary target, range, or visibility rules. A Flying unit being “never Engaged” does not make every attack against or by it automatically legal.

RAW clarification note

Flying ignores Full Cover only. RAW still allows Direct Cover and the Elevation Dead Zone to block visibility against Flying units. Flying also still has an end-position spacing rule: a Flying unit must finish movement at least 1" away from enemy Flying units.

Are the relevant bases within 1" horizontally?

2 • Tags

Are the combat tags compatible, or is one side Flying and therefore never Engaged?

3 • Terrain

Is Size 2+ terrain or another terrain rule blocking engagement through that space?

4 • Elevation

Do the elevation rules actually allow engagement between those positions or access points?

27.1 Close Ranks
Rule text

Before attacking, the unit may make a Close Ranks reposition:

  • choose a leading model
  • move it up to 3" under standard move logic
  • but it must end closer to the enemies it was already engaged with
  • and it cannot use this movement to engage new enemy units.

  • reposition the rest of the unit into coherency

Rule text

The engaged unit attacks enemy units it is engaged with.

Rule text

Melee casualty removal uses a stricter priority system than normal visibility-based casualty handling. The source gives a priority order that protects engagement integrity:

  • remove models not within engagement range of any enemy unit
  • then remove models within engagement range but not in base contact
  • then remove models in base contact.

Mission-Marker Control
Marker control is a legal state, not just a matter of being nearby. Distance matters, but so do line of sight, elevation, coherency, and eligible supply.
Selective RAW lens
Marker control is eligibility first, supply second
RAW says
“A Unit may Contest a Mission Marker only if all three conditions are met …” and “A Unit at Supply 0 may still Control a Marker if no Enemy Units contest it.”
Plain English

You do not start by counting supply. First ask whether the unit is legally eligible to contest at all. Only then do you compare current supply totals.

Table effect

Supply 0 still matters in an uncontested state. Tied supply does not flip control, and once a marker is claimed it stays claimed until the opponent retakes it with a higher eligible total.

Worked ruling
Supply 0 can still matter

A supply-0 unit loses head-to-head comparisons, but it can still control a marker when no enemy unit is legally contesting it. Supply 0 means weak control, not no control.

Overview
Section summary

A unit can contest a mission marker only if:

  • it is on the battlefield
  • at least one model is within 3" of the marker
  • that model has line of sight to the marker
  • that model is on the same elevation as the marker
  • and the unit is in coherency.
  • The marker is treated as Size 0 for LoS purposes.
29.1 Units That Cannot Control
Rule text

Flying units can never control or contest mission markers.

29.2 Supply Decides Control
Rule text

Add up the current supply value of each eligible contesting unit. The player with the higher total controls the marker.

If the totals tie:

  • the marker is contested
  • control does not transfer.
29.3 Sticky Control
Rule text

Once a marker is controlled, it remains under that player’s control until the opponent ends a later scoring phase with a higher eligible contesting supply total. A contested result does not neutralize the marker. A marker cannot revert to neutral once claimed.

29.4 Supply 0 And Control
Rule text

A supply-0 unit may still control a marker if there is no enemy unit contesting it.

RAW references: Parts 8.9.1, 6.2, 12.6.
Lore note

Wars in the sector are fought for more than casualties. Command nodes, warp sites, extraction points, xel’naga ruins, reactor yards, and strategic crossroads all matter. Scenario control rules reflect the fact that commanders in StarCraft win by securing critical objectives, not only by destroying whatever stands in front of them.

RAW clarification note

Supply 0 does not mean "cannot control." RAW only says Supply 0 loses head-to-head comparisons. If no enemy unit is contesting the marker, a Supply 0 unit can still control it.

Decision ladder
Marker control in the right order
Step 1

Check battlefield presence, distance, line of sight, elevation, and coherency.

Step 2

Remove units that are Flying or otherwise ineligible.

Step 3

Add current supply only for the units still left in the contest.

Step 4

Higher eligible total controls. A tie leaves the marker contested and does not neutralize sticky control.

Active Abilities
Active abilities are chosen effects with strict timing. Their power comes from timing discipline, not from being declared whenever they would be most helpful.
Selective RAW lens
“Immediately after” means after the whole action
RAW says
“The ability must be triggered either immediately before declaring an action or immediately after fully resolving one. It cannot be used during an action.”
Plain English

“After” is not a pause inside a charge, a ranged-attack sequence, or a casualty step. It means after the action and its compulsory substeps are finished.

Table effect

Resolve the action completely first. Then declare the Active Ability, unless a more specific rule explicitly opens a special timing window mid-action.

Common misread
"Immediately after" does not mean "pause the middle of the action"

Players often try to slide an Active Ability into the middle of a charge, attack sequence, or casualty step. RAW treats that as too late for "before" and too early for "after" unless a more specific rule opens that timing window.

Overview

Active abilities:

  • require the unit to be currently activated
  • must be used immediately before declaring an action or immediately after fully resolving one
  • cannot be used during the action itself
  • usually expire at the end of the round unless stated otherwise
  • cannot be used in reserves unless the rule says otherwise.

Rule text

A specific unit may use a named active ability only once per round, unless it has REPEATABLE. This is per unit, not global.

RAW clarification note

The timing window is strict. An Active Ability must be used either immediately before declaring an action or immediately after fully resolving one. It cannot be inserted during an action unless the source specifically overrides that rule.

Immediately after fully resolving one means after the whole action is over, including compulsory substeps created by that action. You cannot pause inside a Charge, Ranged Attack sequence, casualty step, or Close Combat sequence to declare an Active Ability unless a more specific rule explicitly creates that timing window.

37.1 BUFF
Rule text

BUFF improves the named characteristic. For target-number characteristics such as Armour or Hit, BUFF reduces the number needed. For value characteristics such as Speed or RoA, BUFF increases the value.

37.2 BURROWED
Rule text

BURROWED is a status with multiple effects, including:

  • gaining HIDDEN
  • size treated as 0
  • current supply treated as 0 for disengage checks
  • cannot control or contest mission markers
  • limited allowed actions while burrowed
  • may still use special abilities unless stated otherwise
  • may make an Evade roll against every attack targeting it
  • other models can move through it under the keyword’s restrictions.
  • This is a strong example of why Part 11 must still be consulted for exhaustive keyword resolution.
37.3 BLOCKING TERRAIN
Rule text

A terrain piece with effective size 1 or greater is blocking terrain for LoS purposes, but whether it actually blocks a given shot still depends on the cover rules.

37.4 ANTI-EVADE and similar modifiers
Rule text

Many weapon keywords apply during a precise step in the attack sequence. Advanced play depends on asking not just “what does it do?” but “when does it do it?”

37.5 PLACE, INSTANT, DISPLACEMENT, AND FLYING
Rule text

These four entries create a large share of table disputes. Read them this way in RAW order:

  • PLACE removes and sets the Leading Model directly. It ignores pathing, Gap Clearance, terrain restrictions, and elevation requirements, but it still must end in a legal position.
  • INSTANT is a keyword effect on weapons and attacks. It is not the same thing as the ordinary lowercase word "instant" used elsewhere in the rules.
  • DISPLACEMENT triggers if the Leading Model ends overlapping the token or model named by that rule. Resolve the overlap immediately by setting the displaced piece into base-to-base if possible, otherwise as close as possible. The source pronouns are clumsy here, so resolve the effect immediately and consistently.
  • FLYING units ignore terrain for movement, are never Engaged, cannot Charge or be Charged, do not participate in the Combat Phase, and cannot control or contest Mission Markers.
RAW references: Part 11, Part 2.6, Parts 7-8.
RAW clarification note

BURROWED permission to use Special Abilities does not override the Combat Phase restriction. If a BURROWED unit begins the Combat Phase Engaged, it must first use Close Ranks to remove BURROWED; if it does not or cannot perform Close Ranks, it does not attack.

RAW clarification note

PLACE is not path movement. It ignores pathing, Gap Clearance, terrain restrictions, and elevation requirements, but it still checks final legality. The unit may be set Within enemy Engagement Range unless the effect says otherwise, and a unit that is PLACED loses STATIONARY because the glossary treats being PLACED as a separate way of changing position.

INSTANT stops Enemy Reaction Abilities to that attack, but it does not skip the normal attack sequence once the Batch is resolving. DISPLACEMENT resolves immediately when the overlap condition is met; do not finish the rest of the move first and then displace later.

FLYING removes engagement and most terrain movement limits, but not ordinary target, range, or visibility requirements. It also ignores Full Cover only; Direct Cover and the Elevation Dead Zone still apply unless a more specific rule says otherwise.

Check yourself • Does a keyword usually replace the whole core sequence?

No. Most keywords replace a narrow part of a sequence, not the whole sequence. The safest read is to identify the exact override and then leave the rest of the normal rules in place.

Overview
Section summary

When a rules dispute happens at the table, use this order:

Step 1 - Identify the action type
Rule text

Is this:

  • army building
  • deployment
  • movement
  • ranged attack
  • charge
  • close combat
  • scoring
  • ability use
  • card-resource payment
  • or keyword/status handling?
Step 2 - Find the general system rule
Rule text

Use the relevant core section first:

  • Parts 3-8 for core battlefield procedures
  • Part 9 for pre-game and list construction
  • Part 10 for ability and card timing
  • Part 11 for keyword text.
Step 3 - Check for a specific override
Rule text

Look for:

  • card text
  • mission text
  • deployment text
  • keyword text
  • special ability text.
  • Specific text overrides the general rule.
Step 4 - Check timing
Rule text

Many arguments are really timing mistakes:

  • Was the unit activated?
  • Was the trigger window missed?
  • Was the reaction already spent this activation?
  • Was the unit even eligible while in reserves?
Step 5 - Check legality constraints
Rule text

Most illegal plays fail because one of these was ignored:

  • line of sight
  • range
  • target tag mismatch
  • engagement restrictions
  • supply limit
  • coherency
  • zone of influence
  • same elevation for marker contesting.
Step 6 - Resolve and move on
Rule text

If the text still does not settle it, use the source quick reference and agree on a consistent interpretation for the current game.
